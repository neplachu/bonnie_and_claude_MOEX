# MOEX SLA Incident Dashboard

Проект для автоматизации анализа SLA-инцидентов в процессах загрузки данных MOEX.

Дашборд показывает не только фактические SLA-просрочки, но и все инциденты загрузки данных:

```text
SLA_FALL = 1 → инцидент привёл к SLA-просрочке
SLA_FALL = 0 → инцидент был, но его устранили до нарушения SLA
```

## Что умеет дашборд

* считает количество инцидентов и SLA-просрочек;
* показывает распределение по доменам, проектам и ETL-объектам;
* анализирует причины сбоев;
* считает время устранения;
* показывает критичность и Risk Score;
* находит повторяющиеся инциденты;
* учитывает зависимости между реквестами;
* показывает возможные root cause и downstream impact.

## Логика данных

Итоговая таблица строится из нескольких источников:

```text
table_bugs.csv          → факты инцидентов
table_domains.csv       → домены и критичность
table_loaders.csv       → статусы загрузочных реквестов
table_dependencies.csv  → зависимости между реквестами
```

Связи:

```text
table_bugs.PID = table_domains.PROJECT_ID
table_bugs.REQ_ID = table_loaders.REQ_ID
table_bugs.REQ_ID = table_dependencies.UP_REQ_ID
```

Для зависимостей используется логика:

```text
I_REQ_ID → UP_REQ_ID
```

Где `I_REQ_ID` — upstream-реквест, а `UP_REQ_ID` — downstream-реквест, который от него зависит.

## Основные метрики

* всего инцидентов;
* SLA-просрочки;
* доля SLA-просрочек;
* Critical + High инциденты;
* Risk Score;
* среднее и P90 время устранения;
* каскадные инциденты;
* downstream impact;
* топ root cause объектов;
* топ проблемных зон.

## Локальный запуск

Установить зависимости:

```bash
pip install -r requirements.txt
```

Запустить приложение:

```bash
python app.py
```

Открыть в браузере:

```text
http://127.0.0.1:8050/
```
